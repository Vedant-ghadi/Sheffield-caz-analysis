## ðŸ”¬ Technical Methodology

To ensure robustness, this analysis utilizes three independent statistical frameworks to isolate the policy effect from natural variation (weather, seasonality, and secular trends).

### 1. Seasonal ARIMA (SARIMA) Forecasting
*   **Objective:** Construct a baseline counterfactual ("business as usual") based on pre-intervention dynamics.
*   **Specification:** `ARIMA(1,1,1)(1,0,0)[7]` with weekly seasonality.
*   **Result:** Actual post-intervention NOâ‚‚ levels diverged **29.9%** below the ARIMA forecast ($RMSE = 8.38$).

### 2. Bayesian Structural Time Series (Prophet)
*   **Objective:** Model complex multi-seasonal patterns (yearly, weekly, daily) using a decomposable time series model.
*   **Specification:** Additive regression model with non-linear trends and holiday effects.
*   **Result:** The model estimates a **44.2% reduction**, suggesting that traditional ARIMA methods may underestimate the effect size when strong seasonalities are present.

### 3. Interrupted Time Series (ITS) Regression
*   **Objective:** Formally test for causal structural breaks using segmented regression.
*   **Model:** $Y_t = \beta_0 + \beta_1 T + \beta_2 D + \beta_3 P + \epsilon_t$
    *   $D$ (Intervention): Immediate level change ($\beta_2 = -2.35$, $p=0.002$)
    *   $P$ (Time since intervention): Slope change ($\beta_3 = -0.014$, $p<0.001$)
*   **Conclusion:** The policy resulted in both an immediate drop and an accelerated rate of improvement.

## ðŸ’» Reproducible Workflow

This repository contains a fully reproducible R pipeline designed for environmental policy auditing.

### Prerequisites
*   **Language**: R (v4.5.2+)
*   **Key Libraries**: `forecast`, `prophet`, `tidyverse`, `zoo`, `lmtest`

### Pipeline Structure
1.  **`scripts/01_extract_openmeteo.R`**: ETL pipeline fetching granular hourly data from CAMS/Open-Meteo.
2.  **`scripts/03_preprocess.R`**: Feature engineering (temporal lags, exogenous regressors).
3.  **`scripts/04_time_series_diagnostics.R`**: Stationarity testing (ADF, KPSS) and STL decomposition.
4.  **`scripts/06_interrupted_time_series.R`**: Econometric regression modeling.

To replicate the study, clone the repository and execute the scripts in numerical order.

```bash
git clone https://github.com/Vedant-ghadi/sheffield-caz-analysis.git
```
